# 产品需求文档 (PRD): 碑文检索系统 - 字段级筛选功能

## 1. 背景与目标 (Background & Objective)
当前系统的检索功能采用"全局模糊匹配"策略，默认在"器名"、"释文"和"出土/发现"中同时搜索。
**痛点**：用户无法精确控制检索范围。例如，当用户只想查找"唐代"（时代）的碑文时，如果"唐代"一词出现在某篇碑文的"释文"中，该碑文也会被检索出来，导致结果不精准（噪音数据）。
**目标**：允许用户手动勾选检索字段（如仅勾选"器名"或"时代"），系统仅在用户指定的字段范围内进行关键词匹配，从而提高检索结果的精确度。

## 2. 核心需求 (Core Requirements)

### 2.1 用户界面 (Frontend UI)
*   **位置**：在搜索框下方或侧边新增"检索范围"选择区。
*   **交互控件**：使用复选框组 (Checkbox Group)，支持多选。
*   **选项列表**：
    *   器名 (Name)
    *   时代 (Era)
    *   别称 (Alias)
    *   释文 (Transcript)
    *   出土/发现 (Discovery)
    *   现藏地点 (Collection)
    *   著录 (Publication)
    *   形制 (Format)
*   **默认状态**：
    *   为保持现有习惯，默认勾选 **"器名"、"释文"、"出土/发现"**（即当前系统的默认搜索范围）。
    *   或者提供一个"全选/重置"按钮。

### 2.2 搜索逻辑 (Search Logic)
*   **输入**：用户输入关键词 `q` (如 "唐") + 用户勾选的字段列表 `fields` (如 ["era", "name"])。
*   **处理规则**：
    1.  **特定字段搜索**：如果用户勾选了字段，系统仅生成针对这些字段的查询条件。
        *   逻辑关系：**OR (或者)**。
        *   *示例*：勾选 "器名" 和 "时代"，搜索 "唐"。
        *   *SQL语义*：`WHERE name LIKE '%唐%' OR era LIKE '%唐%'`。
    2.  **权重排序 (原有逻辑保留)**：即使在筛选模式下，仍应保留"器名匹配优先"的排序逻辑（如果"器名"被包含在筛选范围内）。
    3.  **异常处理**：
        *   如果用户**未勾选任何字段**：
            *   *方案 A (推荐)*：提示用户至少选择一项。
            *   *方案 B*：自动回退到默认范围（器名+释文+出土）。

### 2.3 接口定义 (API Specification)
*   **接口地址**：`GET /api/search`
*   **新增参数**：`fields` (List[str], Optional)
    *   例如：`/api/search?q=唐&fields=name&fields=era`
*   **响应**：返回符合上述逻辑的 `Inscription` 对象列表。

## 3. 用户故事 (User Stories)
1.  **场景一（精准断代）**：
    *   用户只想找"唐代"的碑文，不希望看到释文中提到"唐代"但实际是"宋代"的碑文。
    *   *操作*：取消所有勾选，只勾选"时代"，输入"唐"，点击检索。
    *   *结果*：只返回时代字段包含"唐"的记录。

2.  **场景二（多维查找）**：
    *   用户记得某个碑文可能叫"XX墓志"，或者别称里有"XX"。
    *   *操作*：勾选"器名"和"别称"，输入"XX"。
    *   *结果*：返回器名或别称中包含"XX"的记录。

## 4. 待确认问题 (Questions & Assumptions)
1.  **默认值确认**：是否同意默认勾选 `['name', 'transcript', 'discovery']` 以保持对旧用户的兼容性？
    *   *预设*：是。
2.  **空选行为**：当用户把所有勾选都取消并点击搜索时，系统应报错还是搜索全部？
    *   *预设*：搜索默认核心字段或提示错误。建议提示"请选择至少一个检索范围"。

## 5. 验收标准 (Acceptance Criteria)
1.  前端能够正确展示8个字段的复选框。
2.  勾选不同组合时，后端生成的SQL查询条件正确变化。
3.  未勾选"释文"时，释文中包含关键词但其他字段不包含的记录**不应**出现在结果中。
4.  同时勾选多个字段时，只要任意一个字段匹配即可返回结果。
